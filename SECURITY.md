# Security Policy

## Reporting Security Issues

Please do not open public issues for security vulnerabilities.

Instead, contact:

gondim at gmail.com

(or GitHub/Codeberg private advisory)

## What to Include

- affected version
- reproduction steps
- impact
- proof of concept
- suggested fix (optional)

## Response

I will acknowledge reports as soon as possible.

## Disclosure

I prefer responsible disclosure.

Security issues will be fixed before public disclosure whenever possible.
