# Development Philosophy

> *This document describes how this software is developed, how Artificial Intelligence is used during the engineering process, and where human responsibility remains. It exists as a commitment to transparency for users, contributors, and the Open Source community.*

---

# About Me

My name is **Marcelo Gondim**.

I have more than 30 years of experience working with Internet infrastructure, Linux systems, networking, software engineering, and systems architecture.

Although I continuously learn new programming languages and technologies, software engineering itself is not new to me. My professional experience includes software architecture, algorithms, systems analysis, distributed systems, network protocols, and production infrastructure.

The projects published in this repository are practical software created to solve real-world problems that I encounter in my own infrastructure and in production environments that I support professionally.

Whenever possible, I choose to publish these projects as Open Source because I believe useful software should benefit the broader community.

---

# Why I Publish Open Source Software

I believe software is one of the most effective ways to share knowledge.

Open Source allows ideas, engineering experience, and practical solutions to be shared, improved, and reused by others.

Software engineering is much more than writing source code.

It is the process of transforming knowledge, experience, creativity, and engineering judgment into reliable systems that solve real-world problems.

The value of software is not measured solely by the lines of code it contains, but also by the decisions that define its architecture, maintainability, performance, reliability, security, and long-term evolution.

For this reason, I believe software developers deserve recognition not only for implementing code, but also for the engineering expertise behind every project.

I value openness, interoperability, open standards, and the long-term sustainability of Free and Open Source Software.

These principles closely align with my own professional values.

# Engineering Principles

Every project published here follows a simple principle:

**Engineering decisions are always made by a human.**

Artificial Intelligence is a tool that assists the engineering process—not a replacement for engineering judgment.

Every significant technical decision is evaluated, reviewed, and ultimately made by me.

This includes decisions regarding:

* software architecture;
* project organization;
* algorithms;
* data structures;
* protocol design;
* libraries and dependencies;
* performance;
* security;
* maintainability;
* software quality.

---

# AI-Assisted Development

Artificial Intelligence is used throughout development as an engineering assistant.

Its purpose is to improve productivity, accelerate implementation, assist learning, and encourage technical discussion.

AI may assist by:

* explaining programming concepts;
* discussing language-specific features;
* suggesting implementations;
* reviewing existing code;
* identifying possible bugs;
* proposing optimizations;
* suggesting refactorings;
* comparing alternative approaches;
* helping understand compiler diagnostics;
* automating repetitive tasks.

AI provides suggestions.

It does **not** make engineering decisions.

---

# Development Workflow

Every project follows the same engineering process.

## 1. Problem Definition

Every project begins with a real problem that requires a practical solution.

The objectives, requirements, constraints, and expected behavior are defined by me.

---

## 2. Software Design

The overall architecture is designed before implementation.

This includes decisions regarding:

* system architecture;
* module organization;
* interfaces;
* protocols;
* algorithms;
* libraries;
* data models;
* performance goals.

---

## 3. AI Assistance

Artificial Intelligence may assist in implementing specific components or discussing alternative solutions.

Its role is comparable to documentation, technical books, code examples, or pair programming.

Suggestions are treated as technical input—not authoritative answers.

---

## 4. Technical Review

Every AI-generated suggestion is reviewed before becoming part of the project.

Generated code may be:

* accepted;
* modified;
* optimized;
* refactored;
* rewritten;
* or completely discarded.

There is no automatic acceptance of generated content.

---

## 5. Integration

Accepted implementations are integrated into the overall architecture.

During this stage, code is adapted to maintain consistency, readability, maintainability, and compatibility with the rest of the project.

---

## 6. Validation

Every project is validated through appropriate engineering practices, including:

* compilation;
* testing;
* debugging;
* functional verification;
* performance analysis;
* code review.

Only validated code becomes part of the published software.

---

# Human Responsibility

I assume full responsibility for every project published under my name.

This responsibility includes:

* engineering decisions;
* software architecture;
* implementation choices;
* code review;
* testing;
* debugging;
* maintenance;
* future development.

Artificial Intelligence does not assume responsibility for software.

I do.

---

# My Definition of AI-Assisted Development

For the purposes of this project, I define AI-assisted software development as follows:

> **AI-assisted software development is a process in which Artificial Intelligence provides suggestions, implementations, explanations, and technical assistance while a human engineer remains responsible for the software architecture, engineering decisions, code review, validation, testing, integration, maintenance, and the final published work.**

The defining characteristic is **not** who typed each individual line of source code.

The defining characteristic is **who exercised engineering judgment and accepted responsibility for the final software.**

---

# Transparency

I believe transparency benefits everyone involved in Open Source.

For this reason, I openly acknowledge that Artificial Intelligence is used during the development process.

Using AI does not replace engineering knowledge, critical thinking, testing, or responsibility.

Instead, it is one of many tools available to improve productivity while keeping humans responsible for the software they create.
